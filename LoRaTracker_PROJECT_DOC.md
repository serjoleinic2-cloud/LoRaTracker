## 1. Что это

Мобильное Android-приложение для приёма и визуализации данных с полевых детекторов LoRa32. Детекторы распознают объекты по звуку (INMP441 + FFT на ESP32-C3) и передают телеметрию через LoRa на телефон оператора.

Также приложение может работать в автономном режиме: использовать микрофон и GPS самого телефона как детектор.

## 3. Протокол данных (LoRa32 → Телефон)

Строка CSV через USB-Serial, 115200 baud:

```
Received packet - 'FIELD1;FIELD2;...;FIELD13' with RSSI -57
```

| # | Поле | Пример | Описание |
|---|------|--------|----------|
| 1 | delayMs | 27254 | Задержка отправки |
| 2 | gpsSats | 8 | Количество спутников |
| 3 | latitude | 55.7558 | GPS широта |
| 4 | longitude | 37.6173 | GPS долгота |
| 5 | gpsHour | 14 | Часы UTC |
| 6 | gpsMinute | 30 | Минуты UTC |
| 7 | gpsSecond | 0 | Секунды UTC |
| 8 | accelX | 0.51 | Акселерометр X |
| 9 | accelY | -0.14 | Акселерометр Y |
| 10 | accelZ | 10.27 | Акселерометр Z (≈1g) |
| 11 | temperature | 26.46 | Температура °C |
| 12 | soundPeakFreq | 109.37 | Пиковая частота FFT, Гц |
| 13 | soundCenterFreq | 264.88 | Spectral centroid, Гц |

### RSSI
- Значение после `with RSSI`: -40...-120 dBm
- -50 = близко, -100 = далеко

## 4. Классификация объектов по звуку

| Объект | Пиковая частота | Centroid/Peak | dB мин | Эмодзи |
|--------|-----------------|---------------|--------|--------|
| Человек | 85–350 Гц | >1.5 | >40 | 🚶 |
| Группа людей | 150–500 Гц | >2.0 | >50 | 👥 |
| Легковая машина | 200–1500 Гц | >2.0 | >55 | 🚗 |
| Грузовик / автобус | 80–800 Гц | >1.8 | >65 | 🚚 |
| Мотоцикл | 1000–5000 Гц | >2.5 | >70 | 🏍 |
| Дрон | 8000–20000 Гц | >3.0 | >50 | 🚁 |
| Велосипед | 50–200 Гц | >1.3 | >35 | 🚲 |
| Стройка / техника | 300–2000 Гц | >2.2 | >75 | 🏗 |
| Неопределено | — | — | — | 🌿 |

### Формула классификации:
1. Дрон — приоритет по высоким частотам (>6 кГц)
2. Мотоцикл — высокие обороты (>1 кГц, ratio >2.5)
3. Грузовик — низкий гул (<500 Гц, громко)
4. Легковая — средние частоты
5. Люди — голосовой диапазон

## 5. Железо детектора

| Компонент | Модель | Параметры |
|-----------|--------|-----------|
| Микроконтроллер | ESP32-C3 | RISC-V, WiFi/BT, I2S, UART |
| Микрофон | INMP441 | I2S, 24-бит, 50 Гц–13 кГц, -26 dBFS @ 94 dB SPL |
| GPS | G18U8TTL | u-blox M8030, 72 канала, встроенная антенна |
| Радио | LoRa32 (SX1276) | 433/868 МГц, до 15 км LOS |

### INMP441
- Чувствительность: голос ~5м, мотоцикл ~20м, грузовик ~50м
- Максимум: танк/грузовик на холостом — до 100м
- Ограничение: ветер, дождь создают ложные срабатывания

### G18U8TTL
- Точность: ~2.5м CEP (хороший сигнал)
- Холодный старт: ~27 секунд
- Встроенная антенна — работает только на открытом пространстве

## 8. Режимы работы

### 8.1 Режим USB-LoRa (основной)
- Телефон подключен через OTG к LoRa32
- Получает реальные данные с полевых детекторов
- Может принимать от нескольких детекторов (по очереди)

### 8.2 Режим "Телефон = детектор" (автономный)
- Использует микрофон Samsung S21+ (CAMCORDER source)
- GPS телефона для позиционирования
- FFT на CPU телефона (8192 точек, 48 кГц)
- Полезно для теста и как резервный детектор

### 8.3 Тестовый режим (отключён)
- Генерация фейковых данных для проверки UI
- Больше не используется — заменён реальным микрофоном

## 9. Известные проблемы

| Проблема | Статус | Решение |
|----------|--------|---------|
| Samsung NR (шумоподавление) | Решается | CAMCORDER source + отключение NoiseSuppressor |
| GPS indoors | Ожидаемо | Только на улице, G18U8TTL со встроенной антенной слабый |
| LoRa дальность | Ожидаемо | До 15 км LOS, в городе 1-3 км |
| Потери пакетов | Ожидаемо | Надёжность LoRa ~90%, повторная отправка в прошивке |
